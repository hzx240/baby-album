```prisma
model UploadTask {
  id            String    @id @default(uuid())
  userId        String    @map("user_id")
  familyId      String    @map("family_id")
  childId       String?   @map("child_id")
  status        String    @default("PENDING")
  totalFiles    Int       @default(0) @map("total_files")
  uploadedFiles Int       @default(0) @map("uploaded_files")
  failedFiles   Int       @default(0) @map("failed_files")
  totalBytes    BigInt?   @map("total_bytes")
  uploadedBytes BigInt    @default(0) @map("uploaded_bytes")
  startedAt     DateTime? @map("started_at")
  completedAt   DateTime? @map("completed_at")
  createdAt     DateTime  @default(now()) @map("created_at")
  updatedAt     DateTime  @updatedAt @map("updated_at")

  user  User             @relation("UploadTasks", fields: [userId], references: [id])
  family Family          @relation("UploadTasks", fields: [familyId], references: [id])
  child  Child?          @relation("UploadTasks", fields: [childId], references: [id])
  files UploadTaskFile[]

  @@index([userId])
  @@index([familyId])
  @@index([status])
  @@map("upload_tasks")
}

model UploadTaskFile {
  id             String   @id @default(uuid())
  taskId         String   @map("task_id")
  fileName       String   @map("file_name")
  fileSize       BigInt   @map("file_size")
  checksum       String
  status         String   @default("PENDING")
  retryCount     Int      @default(0) @map("retry_count")
  errorMessage   String?  @map("error_message")
  uploadedBytes  BigInt   @default(0) @map("uploaded_bytes")
  totalChunks    Int      @default(1) @map("total_chunks")
  uploadedChunks Int      @default(0) @map("uploaded_chunks")
  photoId        String?  @map("photo_id")
  createdAt      DateTime @default(now()) @map("created_at")
  updatedAt      DateTime @updatedAt @map("updated_at")

  task         UploadTask    @relation(fields: [taskId], references: [id])
  photo        Photo?        @relation("UploadTaskFiles", fields: [photoId], references: [id])
  chunkUploads ChunkUpload[]

  @@index([taskId])
  @@index([status])
  @@index([checksum])
  @@map("upload_task_files")
}

model ChunkUpload {
  id           String   @id @default(uuid())
  fileRecordId String   @map("file_record_id")
  chunkIndex   Int      @map("chunk_index")
  chunkSize    Int      @map("chunk_size")
  etag         String
  uploadedAt   DateTime @default(now()) @map("uploaded_at")

  fileRecord UploadTaskFile @relation(fields: [fileRecordId], references: [id])

  @@unique([fileRecordId, chunkIndex])
  @@index([fileRecordId])
  @@map("chunk_uploads")
}
```

### 3.3 技术要点

**年龄计算**（精确到天）：
```typescript
function calculateAge(birthDate: Date, currentDate: Date): string {
  const years = currentDate.getFullYear() - birthDate.getFullYear();
  const months = currentDate.getMonth() - birthDate.getMonth();
  const days = currentDate.getDate() - birthDate.getDate();

  if (days < 0) {
    const lastMonth = new Date(currentDate.getFullYear(), currentDate.getMonth(), 0);
    const adjustedDays = lastMonth.getDate() + days;
    return `${years}岁${months - 1}个月${adjustedDays}天`;
  }

  if (months < 0) {
    return `${years - 1}岁${12 + months}个月${days}天`;
  }

  return `${years}岁${months}个月${days}天`;
}
```

**"X年前的今天"查询**：
```sql
-- SQLite查询
SELECT p.*
FROM photos p
JOIN children c ON p.childId = c.id
WHERE
  c.id = :childId
  AND strftime('%m-%d', p.takenAt) = strftime('%m-%d', DATE('now'))
  AND strftime('%Y', p.takenAt) < strftime('%Y', 'now')
ORDER BY p.takenAt DESC;
```

## 4. API命名规范

### 4.1 RESTful风格

- 使用复数名词：`/albums`, `/photos`, `/milestones`
- 使用kebab-case：`/album-photos`, `/upload-tasks`
- 层级关系：`/albums/:id/photos`
- 动作后缀：`/albums/:id/share`, `/upload/:taskId/pause`

### 4.2 统一响应格式

**成功响应**：
```json
{
  "data": { ... },
  "meta": {
    "total": 100,
    "page": 1,
    "limit": 20
  }
}
```

**错误响应**：
```json
{
  "statusCode": 400,
  "message": "错误信息",
  "error": "Bad Request"
}
```

## 5. 数据库迁移工作流程

### 5.1 步骤

1. **编写schema**：在`backend/prisma/schema.prisma`中添加新表
2. **验证schema**：`npx prisma validate`
3. **生成client**：`npx prisma generate`
4. **创建migration**：`npx prisma migrate dev --name description`
5. **应用migration**：`npx prisma migrate deploy`

### 5.2 注意事项

- 使用snake_case命名数据库字段
- 添加适当的索引（外键、查询字段）
- 使用CASCADE删除确保数据一致性
- 添加@map注解指定数据库表名
