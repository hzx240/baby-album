## 🎯 测试系统功能

### ✨ 自动化功能

1. **Git Pre-Commit 钩子**
   - 每次 `git commit` 前自动运行回归测试
   - 测试通过 → 允许提交
   - 测试失败 → 阻止提交并显示错误
   - 防止有问题的代码进入仓库

2. **完整的回归测试套件** (18个测试用例)
   - **用户认证** (4)：注册、登录、获取用户信息、刷新Token
   - **孩子管理** (4)：创建、列表、更新、删除
   - **照片管理** (5)：列表、上传URL、获取URL、删除、验证
   - **家庭成员** (3)：获取成员、创建邀请、获取邀请
   - **性能验证** (2)：并发请求、响应时间

3. **快速验证测试** (3个测试用例)
   - 认证流程验证
   - API 端点连通性
   - 基础功能检查

### 📋 可用的测试命令

```bash
# 完整回归测试（推荐在提交前运行）
npm test

# 快速验证（30秒内完成）
npm run test:quick

# 仅测试认证模块
npm run test:auth

# 仅测试照片功能
npm run test:photos

# 安装 Git 钩子（首次使用时运行一次）
npm run setup
```

## 🚀 使用工作流

### 日常开发流程

```bash
# 1. 拉取最新代码
git pull

# 2. 创建功能分支
git checkout -b feature/new-function

# 3. 开发功能
# ... 编写代码 ...

# 4. 提交前运行测试
npm test

# 5. 测试通过后提交（pre-commit 钩子会自动验证）
git add .
git commit -m "feat: 添加新功能"

# 如果测试失败，pre-commit 会阻止提交
# 修复错误后再次运行 npm test
```

### 紧急修复流程

```bash
# 跳过测试直接提交（紧急情况）
git commit --no-verify -m "fix: 修复紧急问题"

# 修复后立即测试
npm run test:quick
```
