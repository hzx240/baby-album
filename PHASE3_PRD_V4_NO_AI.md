## 1. 功能需求详细规格

### 1.1 成长追踪模块 (44h)

#### 1.1.1 成长曲线可视化 (16h)

**功能描述**: 为每个孩子记录身高、体重、头围等生长指标，生成可视化成长曲线。

**用户故事**:
- 作为家长，我希望能够记录孩子的身高体重数据
- 作为家长，我希望看到孩子的成长曲线图，了解生长趋势
- 作为家长，我希望能够对比 WHO 标准生长曲线

**功能要求**:
1. **数据录入**:
   - 支持身高 (cm)、体重 (kg)、头围 (cm) 指标
   - 支持日期选择 (默认当天)
   - 支持批量导入 (CSV 格式)
   - 数据验证: 身高 0-200cm，体重 0-100kg

2. **曲线展示**:
   - 折线图展示时间序列数据
   - 支持 WHO 标准曲线对比
   - 支持多指标同时展示 (身高/体重)
   - 支持日期范围筛选 (最近 1个月/3个月/6个月/1年/全部)

3. **数据管理**:
   - 支持编辑/删除历史记录
   - 支持导出为 CSV/PDF

**技术实现**:
- 前端: Recharts 图表库
- 后端: REST API CRUD
- 数据库: 新增 `GrowthRecord` 模型

---

#### 1.1.2 成长报告生成 (16h)

**功能描述**: 基于成长数据生成可视化报告，支持 PDF 导出。

**用户故事**:
- 作为家长，我希望能够生成孩子的成长报告，分享给家人
- 作为家长，我希望报告包含关键数据和图表

**功能要求**:
1. **报告内容**:
   - 基本信息卡片 (姓名、年龄、性别)
   - 成长曲线图表 (身高/体重)
   - 关键里程碑时间线
   - 最新照片展示 (最近 6 张)
   - 数据摘要 (当前身高体重、生长速度)

2. **模板支持**:
   - 默认模板 (简洁风格)
   - 节日模板 (生日、新年等)
   - 自定义模板 (用户上传背景图)

3. **导出选项**:
   - PDF 格式 (高清)
   - 图片格式 (PNG/JPEG)
   - 分享到社交平台

**技术实现**:
- 前端: jsPDF 库 + Canvas 渲染
- 后端: PDF 生成服务
- 存储: 报告文件存储到 MinIO/S3

---

#### 1.1.3 里程碑提醒 (12h)

**功能描述**: 当孩子接近重要的发育里程碑时，自动提醒家长关注。

**用户故事**:
- 作为家长，我希望在合适的时候收到提醒，记录孩子的里程碑
- 作为家长，我希望了解同龄孩子的发育标准

**功能要求**:
1. **智能提醒**:
   - 基于 WHO 发育里程碑数据
   - 提前 7 天提醒 (如: "您的宝宝 6 个月了，可以尝试坐立了")
   - 支持自定义提醒日期

2. **里程碑库**:
   - 预置 WHO 发育里程碑 (0-5 岁)
   - 分类: 大运动、精细动作、语言、社交
   - 支持手动记录里程碑

3. **通知渠道**:
   - 应用内通知
   - Email 邮件提醒
   - 推送通知 (Phase 4)

**技术实现**:
- 后端: 定时任务扫描里程碑
- 数据: 预置 WHO 里程碑数据
- 通知: 邮件服务 (Phase 1 已有)

### 1.2 社交互动模块 (36h)

#### 1.2.1 照片评论系统 (20h)

**功能描述**: 用户可以对照片进行评论，增强家庭互动。

**用户故事**:
- 作为家庭成员，我希望能够对照片发表评论
- 作为家长，我希望看到其他家庭成员的评论

**功能要求**:
1. **评论功能**:
   - 支持文字评论 (1-500 字符)
   - 支持 Emoji 表情回复
   - 支持评论点赞
   - 支持 @提及其他用户

2. **评论管理**:
   - 支持编辑/删除自己的评论
   - 支持举报不当评论
   - 支持评论通知 (Email/应用内)

3. **展示方式**:
   - 时间线式展示
   - 最新评论置顶
   - 支持折叠/展开

**技术实现**:
- 前端: CommentList 组件 + EmojiPicker 组件
- 后端: REST API CRUD + WebSocket 实时通知
- 数据库: 新增 `PhotoComment` 模型

---

#### 1.2.2 社交分享优化 (16h)

**功能描述**: 优化相册分享功能，支持更灵活的权限控制。

**用户故事**:
- 作为家长，我希望能够分享相册给亲友查看
- 作为家长，我希望能够控制分享的权限和期限

**功能要求**:
1. **分享方式**:
   - 公开链接 (无需登录)
   - 私密分享 (需要密码)
   - 有效期设置 (1天/7天/30天/永久)

2. **权限控制**:
   - 仅查看
   - 可评论
   - 可下载

3. **分享统计**:
   - 访问次数
   - 访客人数
   - 评论数量

**技术实现**:
- 后端: 分享 Token 生成机制
- 数据库: 扩展 `AlbumShare` 模型
- 前端: ShareDialog 分享对话框

### 1.3 智能相册增强 (40h)

#### 1.3.1 基于规则的智能相册 (40h)

**功能描述**: 通过用户自定义规则自动组织照片，无需 AI 算法。

**用户故事**:
- 作为家长，我希望能够根据规则自动创建相册
- 作为家长，我希望规则简单易懂，易于配置

**功能要求**:
1. **规则引擎**:
   - 时间范围 (如: 2024年所有照片)
   - 标签过滤 (如: 包含"生日"标签)
   - 人物过滤 (如: 包含特定家庭成员)
   - 位置过滤 (如: 拍摄于"公园")
   - 组合规则 (AND/OR 逻辑)

2. **自动更新**:
   - 新照片自动匹配规则
   - 相册封面自动更新
   - 支持"自动整理"开关

3. **预设模板**:
   - 按月自动相册 (如: "2024年1月")
   - 生日相册 (如: "宝宝3岁生日")
   - 节日相册 (如: "春节2024")

**技术实现**:
- 前端: SmartRuleBuilder 组件 (已有，需增强)
- 后端: 规则解析引擎 + 定时任务
- 数据库: `Album.smartRules` 字段存储规则 JSON

### 2.3 数据库设计

#### 新增模型

```prisma
model GrowthRecord {
  id          String   @id @default(uuid())
  childId     String   @map("child_id")
  recordType  String   @map("record_type")  // HEIGHT | WEIGHT | HEAD_CIRCUMFERENCE
  value       Float
  date        DateTime
  notes       String?
  createdAt   DateTime @default(now()) @map("created_at")

  child       Child    @relation(fields: [childId], references: [id], onDelete: Cascade)

  @@index([childId, date(sort: Desc)])
  @@map("growth_records")
}

model PhotoComment {
  id          String   @id @default(uuid())
  photoId     String   @map("photo_id")
  userId      String   @map("user_id")
  content     String
  emojiReaction String? @map("emoji_reaction")
  parentId    String?  @map("parent_id")  // 支持评论回复
  likes       Int      @default(0)
  createdAt   DateTime @default(now()) @map("created_at")
  updatedAt   DateTime @updatedAt @map("updated_at")

  photo       Photo    @relation(fields: [photoId], references: [id], onDelete: Cascade)
  user        User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  parent      PhotoComment? @relation("CommentReplies", fields: [parentId], references: [id], onDelete: Cascade)
  replies     PhotoComment[] @relation("CommentReplies")

  @@index([photoId, createdAt(sort: Desc)])
  @@index([userId])
  @@map("photo_comments")
}

model AlbumShare {
  id          String   @id @default(uuid())
  albumId     String   @map("album_id")
  shareToken  String   @unique @map("share_token")
  password    String?  // 可选密码保护
  expiresAt   DateTime? @map("expires_at")
  permissions String   @default("VIEW") // VIEW | COMMENT | DOWNLOAD
  viewCount   Int      @default(0) @map("view_count")
  createdAt   DateTime @default(now()) @map("created_at")

  album       Album    @relation(fields: [albumId], references: [id], onDelete: Cascade)

  @@index([shareToken])
  @@index([expiresAt])
  @@map("album_shares")
}
```

## 3. UI/UX 设计需求

### 3.1 成长曲线页面

**布局**:
- 顶部: 孩子选择器 + 时间范围筛选器
- 中部: 图表区域 (身高/体重折线图)
- 底部: 数据录入按钮 + 历史记录列表

**交互**:
- 点击数据点显示详情
- 长按数据点显示编辑/删除菜单
- 图表支持缩放 (捏合手势)

**设计规范**:
- 主色调: 蓝色 (#3B82F6) 表示男孩，粉色 (#EC4899) 表示女孩
- 曲线样式: 实线 + 圆点标记
- WHO 标准曲线: 虚线 + 灰色

---

### 3.2 评论组件

**布局**:
- 照片下方: 评论列表 (最新 3 条，点击"查看全部"展开)
- 评论输入框: 固定在底部
- Emoji 选择器: 点击笑脸图标弹出

**交互**:
- 点击评论触发点赞动画
- 点击回复切换到回复模式
- 支持 @提及 (输入 @ 触发用户选择)

**设计规范**:
- 评论气泡: 左对齐，圆角
- 自己的评论: 右对齐，蓝色背景
- Emoji 选择器: 8x8 网格布局

---

### 3.3 分享对话框

**布局**:
- 分享方式选择 (公开/私密)
- 权限设置 (查看/评论/下载)
- 有效期选择 (1天/7天/30天/永久)
- 分享链接展示 + 二维码

**交互**:
- 一键复制链接
- 下载二维码图片
- 撤销分享按钮

**设计规范**:
- 主要操作: 蓝色按钮
- 危险操作: 红色按钮 (撤销分享)
