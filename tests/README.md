# 回归测试套件

宝贝成长相册的自动化回归测试脚本，确保每次代码提交前所有基本功能正常工作。

## 📋 目录

```
tests/
├── regression-test.js    # 主测试脚本
└── README.md             # 本文档
```

## 🚀 快速开始

### 安装 Git 钩子（推荐）

```bash
node scripts/setup-hooks.js
```

### 手动运行测试

```bash
# 完整测试套件
npm test

# 快速模式（跳过性能测试）
npm run test:quick

# 仅测试认证
npm run test:auth

# 仅测试照片功能
npm run test:photos
```

## 🔄 Git 钩子工作流程

```bash
# 1. 修改代码
git add .

# 2. 提交（自动运行测试）
git commit -m "feat: 添加新功能"

# 钩子会自动：
# - 运行 node tests/regression-test.js
# - 测试通过 → 允许提交
# - 测试失败 → 阻止提交，显示错误
```

## 🛠️ 故障排除

### 测试失败：退出码 1
1. 检查后端是否运行：`cd backend && npm run start:dev`
2. 检查前端是否运行：`cd frontend && npm run dev`
3. 查看具体错误消息

### 钩子未生效
1. 手动运行：`node scripts/setup-hooks.js`
2. 检查 `.git/hooks/pre-commit` 文件是否存在且有执行权限

## 📝 测试数据说明

- 每次运行会创建新的测试用户（随机邮箱）
- 测试数据会在测试完成后保留
- 可以手动通过浏览器访问应用验证测试数据

## 🎯 最佳实践

1. **提交前测试**：每次提交前运行完整测试套件
2. **功能开发后测试**：开发新功能后立即运行相关测试
3. **重构后测试**：重构代码后运行完整套件防止回归
4. **定期测试**：定期运行测试确保所有功能正常
