### 决策 2: AI 场景分类如何实现？

| 选项 | 成本 | 准确率 | 隐私风险 | 推荐度 |
|------|------|--------|----------|--------|
| **A. TensorFlow.js 本地** | $0 | 80-85% | 无 | ⭐⭐⭐⭐⭐ |
| **B. 云端 API + 合规** | $100-200/月 | 95%+ | 需合规 | ⭐⭐⭐ |

**选项 A: TensorFlow.js 本地** (产品经理推荐)
- ✅ **优点**: 无隐私风险、零 API 成本
- ✅ COPPA/GDPR/PIPL 完全合规
- ❌ **缺点**: 准确率可能较低 (80-85%)
- 工时: +8h

**选项 B: 云端 API + 母亲同意**
- ✅ **优点**: 准确率高 (95%+)
- ❌ **缺点**: 需要 COPPA 合规、API 成本
- 工时: +16h

**混合方案** (产品经理推荐):
- 默认: TensorFlow.js 本地 AI
- 付费: 可选云端 API (需父母同意)
- 成本可控

### 决策 3: PhotoFace 是否存储特征向量？

| 方案 | 隐私风险 | 合规风险 | 推荐度 |
|------|----------|----------|--------|
| **A. 不存储特征向量** | 无 | 无 | ⭐⭐⭐⭐⭐ |
| **B. 加密存储** | 中 | 中 | ⭐⭐⭐ |
| **C. 明文存储** | 高 | 高 | ❌ |

**产品经理建议**: ✅ **方案 A - 不存储特征向量**

**实施方案**:
- 人脸识别仅在用户设备本地处理
- 仅存储用户自定义的人名-标签映射
- 不存储生物识别特征向量

**好处**:
- ✅ 无隐私风险
- ✅ 无合规风险
- ✅ 零额外成本

**限制**:
- 换设备需要重新识别
- 准确率依赖设备性能

### AI 技术栈

| 功能 | 技术选择 | 理由 |
|------|----------|------|
| **场景分类** | TensorFlow.js | 隐私合规、零成本 |

| **自动标签建议** | TensorFlow.js | 隐私合规、零成本 |
| **人脸识别** | 本地设备 | 无隐私风险 |

- ✅ 完全符合 COPPA/GDPR/PIPL
- ✅ 零 API 成本
- ✅ 数据不离开设备
- ✅ 更好的用户体验 (更快的响应)

**劣势**:
- ❌ 模型文件较大 (~20MB)
- ❌ 需要用户设备性能
- ❌ 准确率可能略低 (5-10%)
