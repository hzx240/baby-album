## 📋 执行摘要

本文档为 Phase 3 v4.0（无 AI 版本）制定详细的验收标准。所有功能必须满足以下标准才能通过验收。

### 验收原则

1. **功能完整性**: 所有 Must Have 功能必须完整实现
2. **质量标准**: 测试覆盖率达标，无 P0/P1 缺陷
3. **性能指标**: 满足技术性能要求
4. **安全合规**: 通过安全审计
5. **用户体验**: UI/UX 符合设计规范

## 1. 成长追踪模块验收标准

### 1.1 成长曲线可视化 (P0 - Must Have)

#### 功能验收标准

**AC-1.1.1: 数据录入**
- [ ] 支持录入身高（cm）、体重（kg）、头围（cm）
- [ ] 身高范围验证：0-200cm，精度 0.1cm
- [ ] 体重范围验证：0-100kg，精度 0.01kg
- [ ] 头围范围验证：0-60cm，精度 0.1cm
- [ ] 日期选择器默认当天，可选择历史日期
- [ ] 同一天同一指标只能有一条记录（唯一性约束）
- [ ] 支持添加备注（可选，最多 500 字符）
- [ ] 表单验证：至少填写一项指标
- [ ] 保存成功后显示成功提示

**AC-1.1.2: 曲线展示**
- [ ] 使用 Recharts 折线图展示
- [ ] X 轴：时间（月龄/年龄）
- [ ] Y 轴：指标数值
- [ ] 支持同时展示身高和体重（双 Y 轴）
- [ ] 数据点标记清晰，可点击查看详情
- [ ] 支持 WHO 标准曲线对比（P3, P15, P50, P85, P97）
- [ ] WHO 曲线使用虚线，用户数据使用实线
- [ ] 支持日期范围筛选：1个月/3个月/6个月/1年/全部
- [ ] 图表响应式设计，移动端友好
- [ ] 空数据状态：显示引导录入提示

**AC-1.1.3: 数据管理**
- [ ] 历史记录列表展示（按日期倒序）
- [ ] 支持编辑历史记录
- [ ] 支持删除历史记录（需二次确认）
- [ ] 支持导出为 CSV 格式
- [ ] CSV 包含：日期、身高、体重、头围、备注
- [ ] 支持批量导入 CSV（格式验证）

#### 技术验收标准

**AC-1.1.4: API 实现**
- [ ] `POST /api/children/:id/growth` - 创建成长记录
- [ ] `GET /api/children/:id/growth` - 获取成长记录列表（分页）
- [ ] `GET /api/children/:id/growth/chart` - 获取图表数据
- [ ] `PUT /api/children/:id/growth/:recordId` - 更新成长记录
- [ ] `DELETE /api/children/:id/growth/:recordId` - 删除成长记录
- [ ] `GET /api/children/:id/growth/export` - 导出 CSV
- [ ] `POST /api/children/:id/growth/import` - 导入 CSV
- [ ] 所有 API 需要认证和家庭权限验证
- [ ] API 响应时间 < 500ms（P95）

**AC-1.1.5: 数据库实现**
- [ ] 创建 `GrowthRecord` 模型
- [ ] 字段：id, childId, recordDate, height, weight, headCirc, notes, createdAt, updatedAt
- [ ] 唯一约束：(childId, recordDate)
- [ ] 索引：(childId, recordDate DESC)
- [ ] 级联删除：删除 Child 时删除所有 GrowthRecord

**AC-1.1.6: 测试覆盖**
- [ ] 单元测试覆盖率 > 80%
- [ ] 集成测试：CRUD 操作
- [ ] E2E 测试：完整录入和查看流程
- [ ] 边界测试：数值范围验证
- [ ] 性能测试：1000 条记录加载 < 2s
