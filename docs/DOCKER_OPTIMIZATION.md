# Docker 镜像优化报告

## 优化概览

本次优化针对后端和前端的Dockerfile进行了全面改进，主要目标是减少镜像大小、提升构建速度和增强安全性。

## 后端 Dockerfile 优化

### 优化措施

1. **添加构建参数和标签**
   - 添加 `BUILD_DATE`、`VCS_REF` 等构建参数
   - 使用 OCI 标准标签，便于镜像管理和追踪

2. **优化依赖安装**
   - 使用 `npm ci --prefer-offline --no-audit` 加速安装
   - 添加 `npm cache clean --force` 清理缓存
   - 合并 RUN 命令减少镜像层数

3. **优化系统包安装**
   - 合并 apk 命令并清理缓存
   - 减少不必要的系统依赖

4. **优化文件复制**
   - 使用 `--chown` 参数直接设置文件所有者
   - 减少额外的 `chown` 命令

5. **添加 Node.js 内存限制**
   - 设置 `NODE_OPTIONS="--max-old-space-size=512"`
   - 防止内存溢出

### 预期效果

- 镜像大小减少: ~15-20%
- 构建时间减少: ~10-15%
- 安全性提升: 添加标签和版本追踪

## 前端 Dockerfile 优化

### 优化措施

1. **添加构建参数和标签**
   - 与后端保持一致的标签规范

2. **优化依赖安装**
   - 使用 `npm ci --prefer-offline --no-audit`
   - 清理 npm 缓存

3. **优化健康检查**
   - 简化健康检查命令
   - 减少不必要的路径检查

4. **优化系统包安装**
   - 清理 apk 缓存

### 预期效果

- 镜像大小减少: ~10-15%
- 构建时间减少: ~5-10%

## Redis 配置优化

### 新增配置文件

创建了专用的 Redis 配置文件 `backend/config/redis.conf`，包含：

1. **持久化策略**
   - RDB: 多级保存策略
   - AOF: everysec 模式
   - 混合持久化: aof-use-rdb-preamble

2. **内存管理**
   - maxmemory: 200mb
   - maxmemory-policy: allkeys-lru
   - maxmemory-samples: 5

3. **性能优化**
   - tcp-backlog: 511
   - tcp-keepalive: 300
   - slowlog 配置

4. **安全配置**
   - protected-mode
   - 密码通过环境变量设置

### docker-compose 集成

更新了 `docker-compose.yml` 中的 Redis 配置：
- 添加持久化参数
- 添加性能调优参数
- 添加慢查询日志配置
