## 执行摘要

基于Phase 1和Phase 2的成功交付，Phase 3将聚焦于**智能化增强**和**成长记录工具**两大核心方向，通过AI技术提升用户体验，打造差异化的产品竞争力。

### 核心目标
- 提升用户留存率30%
- 增加用户日均使用时长15分钟
- 建立产品核心护城河（AI能力）
- 拓展商业化路径

## 1. 市场与竞品分析

### 1.1 国内竞品分析

| 产品 | 核心定位 | 核心功能 | 差异化机会 |
|------|----------|----------|------------|
| **亲宝宝** | 家庭云空间 | 云相册、成长记录、社区互动 | AI深度整合不足 |
| **宝宝树** | 育儿知识社区 | 专家问答、育儿工具、成长记录 | 工具属性重，体验一般 |
| **育学园** | 科学育儿 | 专业内容、健康记录 | 内容驱动，记录功能弱 |

### 1.2 国际竞品分析

| 产品 | 核心定位 | 核心功能 | 差异化机会 |
|------|----------|----------|------------|
| **Google Photos** | 通用相册 | AI分类、搜索、共享动画 | 缺乏育儿场景化功能 |
| **Tinybeans** | 家庭日记 | 成长时间线、亲密分享 | AI能力弱，功能单一 |
| **FamilyAlbum** | 家庭共享 | 实时同步、免费打印 | 成长记录功能有限 |
| **23snaps** | 隐私相册 | 多代同堂、时光胶囊 | 无智能分析 |

### 1.3 市场趋势洞察 (2026)

1. **AI原生化**: 用户期待AI主动提供价值，而非被动响应
2. **隐私优先**: 数据安全成为选择产品的首要因素
3. **情感连接**: 数字回忆的承载形式趋于视频化和互动化
4. **跨代协作**: 祖辈参与度提升，需要简化操作
5. **成长可视化**: 数据驱动的育儿决策支持

## 3. Phase 3 功能规划

### 3.1 模块一：AI智能增强

#### 功能 3.1.1: AI照片质量评分

**用户故事**
> 作为父母，我希望系统帮我自动识别照片质量，以便快速筛选出最值得保留的照片。

**功能描述**
- 自动分析照片的技术质量（清晰度、曝光、构图）
- 分析照片的情感价值（笑容、眼神、表情）
- 综合评分：0-100分
- 可按质量排序/筛选

**验收标准**
- [ ] 照片上传后30秒内完成评分
- [ ] 评分准确率 >85%（用户反馈验证）
- [ ] 显示评分等级（优秀/良好/一般/建议删除）
- [ ] 提供"一键筛选最佳照片"功能

**技术方案**
- 集成AWS Rekognition / Google Vision API
- 预计算评分，存储在Photo表新增字段
- 异步任务队列处理，避免阻塞上传

---

#### 功能 3.1.2: 智能场景分类

**用户故事**
> 作为用户，我希望系统能自动识别照片场景（如"第一次走路"、"生日派对"），以便按主题回忆。

**功能描述**
- 自动识别照片中的场景和活动
- 预设场景类型：生日、节日、户外、室内、吃饭、睡觉、玩耍等
- 支持自定义场景标签
- 可按场景筛选照片

**验收标准**
- [ ] 识别准确率 >80%
- [ ] 支持15+种预设场景
- [ ] 支持用户纠正反馈（持续学习）
- [ ] 场景标签可搜索

**技术方案**
```typescript
// 新增表: photo_scenes
model PhotoScene {
  id        String   @id @default(uuid())
  photoId   String   @map("photo_id")
  sceneType String   @map("scene_type") // BIRTHDAY, OUTDOOR, EATING, etc.
  confidence Float
  photo     Photo    @relation(fields: [photoId], references: [id])
}
```

---

#### 功能 3.1.3: 照片去重增强

**用户故事**
> 作为用户，我希望系统能识别相似照片（连续拍摄），并建议保留哪张。

**功能描述**
- 基于视觉相似度识别连续拍摄
- 基于 EXIF 时间识别连拍
- 智能推荐：保留质量最高/表情最好的一张
- 一键处理所有重复组

**验收标准**
- [ ] 相似度检测准确率 >90%
- [ ] 连拍组识别准确率 >95%
- [ ] 推荐保留照片用户采纳率 >80%
- [ ] 支持批量处理

**技术方案**
- 使用 perceptual hash 算法 (pHash) 计算图片指纹
- Hamming distance < 5 视为相似
- 连拍检测：时间差 < 3秒 + 相似度高

#### 功能 3.1.4: 智能照片合集

**用户故事**
> 作为用户，我希望系统能自动将某个时期的照片整理成精美合集，便于回忆。

**功能描述**
- 按时间自动生成月度/季度合集
- 按主题自动生成事件合集
- AI生成合集标题和描述
- 支持一键分享合集

**验收标准**
- [ ] 每月自动生成月度合集
- [ ] AI标题用户满意度 >75%
- [ ] 合集支持预览和编辑
- [ ] 合集可一键分享

**用户故事**

- 记录里程碑事件的生长数据

**验收标准**
- [ ] 支持多个宝贝分别记录
- [ ] 曲线图支持缩放查看
- [ ] 数据可导出
- [ ] 提供"发育正常"提示

**技术方案**
```typescript
// 新增表: growth_records
model GrowthRecord {
  id           String   @id @default(uuid())
  childId      String   @map("child_id")
  recordDate   DateTime @map("record_date")
  recordType   String   @map("record_type") // HEIGHT, WEIGHT, HEAD_CIRCUMFERENCE
  value        Float
  unit         String   // cm, kg
  notes        String?
  recordedBy   String   @map("recorded_by")
  createdAt    DateTime @default(now()) @map("created_at")
  child        Child    @relation(fields: [childId], references: [id])

  @@unique([childId, recordDate, recordType])
  @@index([childId, recordDate])
  @@map("growth_records")
}
```

**依赖库**
- 前端: Chart.js / Recharts (曲线图)
- 数据: WHO Growth Standards 数据集
