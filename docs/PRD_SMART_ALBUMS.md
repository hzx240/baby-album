## 📋 文档概述

### 产品目标
提供智能相册功能，帮助用户更好地组织和分享照片，解决照片查找困难的核心痛点。

### 用户价值
- **组织效率**: 照片组织性提升60%
- **查找便利**: 快速找到特定时刻的照片
- **分享便捷**: 一键分享相册给家人朋友
- **情感价值**: 按事件回忆美好时刻

### 业务价值
- **活跃度提升**: 用户更常查看和分享照片
- **留存率提升**: 相册功能增强粘性
- **口碑传播**: 分享功能带来新用户

## 1. 用户故事

### 主要用户故事

**故事 #1: 自动创建相册**
> 作为用户，我希望上传照片后系统自动按日期创建相册（如"2026年2月13日"），这样我就不需要手动整理照片了。

**验收标准**:
- [ ] 上传照片后自动按拍摄日期创建相册
- [ ] 相册名称为"YYYY年MM月DD日"
- [ ] 同一天的照片自动归入同一相册
- [ ] 用户可以关闭自动创建功能

---

**故事 #2: 手动创建相册**
> 作为用户，我希望可以手动创建相册（如"宝宝满月"、"第一次去公园"），并将照片添加进去，以便按事件组织照片。

**验收标准**:
- [ ] 可以创建自定义相册
- [ ] 可以添加多张照片到相册
- [ ] 可以从相册移除照片
- [ ] 可以重命名相册
- [ ] 可以删除相册

---

**故事 #3: 相册封面**
> 作为用户，我希望可以为相册选择封面照片，这样相册列表更好看。

**验收标准**:
- [ ] 创建相册时自动选择第一张照片作为封面
- [ ] 可以更换相册封面
- [ ] 封面在相册列表中显示

---

**故事 #4: 相册分享**
> 作为用户，我希望可以分享相册给家人朋友，让他们也能查看这些照片。

**验收标准**:
- [ ] 可以生成相册分享链接
- [ ] 可以生成相册分享二维码
- [ ] 可以设置分享过期时间（1天/7天/永久）
- [ ] 可以取消分享
- [ ] 分享链接可以在无登录情况下访问

---

**故事 #5: 相册管理**
> 作为用户，我希望可以管理相册（重命名、删除、合并），以便更好地组织照片。

**验收标准**:
- [ ] 可以重命名相册
- [ ] 可以删除相册（照片不删除）
- [ ] 可以合并多个相册
- [ ] 可以查看相册详情（照片数量、创建时间）

#### 功能 6: 自动创建相册

**需求描述**: 上传照片时自动按日期创建相册

**实现逻辑**:
1. 用户上传照片
2. 系统读取照片的拍摄日期（EXIF）
3. 检查是否存在该日期的相册
4. 如果不存在，自动创建
5. 将照片添加到相册

**功能要求**:
- [ ] 上传照片时自动创建相册
- [ ] 相册名称格式:"YYYY年MM月DD日"
- [ ] 读取EXIF拍摄日期
- [ ] 如果没有EXIF，使用上传日期
- [ ] 用户可在设置中关闭

### 2.3 数据库设计

#### 表 1: albums (相册表)

```sql
CREATE TABLE albums (
  id UUID PRIMARY KEY,
  familyId UUID NOT NULL,
  name VARCHAR(255) NOT NULL,
  description TEXT,
  coverPhotoId UUID,
  isAutoGenerated BOOLEAN DEFAULT false,
  autoGeneratedDate DATE, -- 自动生成的日期
  shareToken VARCHAR(255), -- 分享token
  shareExpiresAt TIMESTAMP, -- 分享过期时间
  createdAt TIMESTAMP,
  updatedAt TIMESTAMP
);

CREATE INDEX idx_albums_family ON albums(familyId);
CREATE INDEX idx_albums_share ON albums(shareToken);
CREATE INDEX idx_albums_auto_date ON albums(autoGeneratedDate);
```

---

#### 表 2: album_photos (相册-照片关联表)

```sql
CREATE TABLE album_photos (
  albumId UUID REFERENCES albums(id) ON DELETE CASCADE,
  photoId UUID REFERENCES photos(id) ON DELETE CASCADE,
  orderIndex INTEGER, -- 照片在相册中的顺序
  addedAt TIMESTAMP,
  PRIMARY KEY (albumId, photoId)
);

CREATE INDEX idx_album_photos_album ON album_photos(albumId);
CREATE INDEX idx_album_photos_photo ON album_photos(photoId);
```

---

#### 表 3: album_share_stats (分享统计表)

```sql
CREATE TABLE album_share_stats (
  id UUID PRIMARY KEY,
  albumId UUID REFERENCES albums(id) ON DELETE CASCADE,
  shareToken VARCHAR(255),
  viewCount INTEGER DEFAULT 0,
  lastViewedAt TIMESTAMP,
  createdAt TIMESTAMP
);

CREATE INDEX idx_share_stats_token ON album_share_stats(shareToken);
```
