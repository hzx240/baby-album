## 2. P0 安全风险详解

### 2.1 风险1：COPPA合规问题 - 儿童在线隐私保护法

**严重程度:** 🔴 P0 - CRITICAL

**问题描述:**
- Phase 2 已实现PhotoFace模型（人脸特征向量存储）
- 美国COPPA（儿童在线隐私保护法）要求：
  - 儿童数据必须获得明确同意
  - 人脸数据必须加密存储
  - 用户可要求删除数据
- 违规最高罚款：**年营业额4%或2000万欧元**

**当前问题:**
- ❌ 没有同意机制
- ❌ 人脸数据可能明文存储
- ❌ 没有删除功能
- ❌ 没有隐私政策

**法律要求:**
```
英国 GDPR:
- 儿童数据：明确同意 + 可撤销
- 加密存储：AES-256
- 数据最小化：仅存储必要信息
- 保留期限：合理时间

美国 COPPA:
- 儿童隐私：需父母同意
- 加密存储：至少256位
- 数据访问控制：严格限制

中国网络安全法:
- 儿童信息：单独同意
- 加密存储：国密算法
- 数据出境：安全评估
```

**工作量:** 16小时

**解决方案:**
1. ✅ 添加同意机制
   - 上传照片前显示同意提示
   - 每年/每个儿童记录一次同意
2. ✅ 实施加密存储
   - 人脸向量AES-256加密
   - 密钥管理服务
3. ✅ 添加删除功能
   - 用户可删除人脸数据
   - 自动删除退出用户数据（30天）
4. ✅ 隐私政策
   - 详细的隐私政策页面
   - 符合GDPR/COPPA要求

**测试用例:** TBD（新增）
- TC-COPPA-001: 同意机制测试
- TC-COPPA-002: 加密存储测试
- TC-COPPA-003: 数据删除测试
- TC-COPPA-004: 隐私政策测试

### 2.2 风险2：AI API 成本失控

**严重程度:** 🔴 P0 - CRITICAL

**问题描述:**
- Phase 3包含大量AI功能（质量评分、场景分类、自动标签）
- AWS Rekognition / Google Vision API按调用计费
- 1000用户，100张照片/用户 = 100,000张照片
- 成本估算：
  - AWS Rekognition: $0.001/图
  - Google Vision: $0.0015/图
  - **月成本可能高达 $2000-4000**（失控风险）

**当前问题:**
- ❌ 无用量监控
- ❌ 无成本上限
- ❌ 无自动暂停
- ❌ 用户配额控制

**工作量:** 12小时

**解决方案:**
1. ✅ 实施缓存策略
   - Redis缓存7天（避免重复调用）
   - 智能预加载
2. ✅ 用量监控
   - Prometheus监控API调用
   - Grafana可视化
   - 每日成本报告
3. ✅ 成本上限
   - 日预算：$500/天
   - 超过90%自动暂停
   - 告警通知
4. ✅ 用户配额
   - 免费用户：10张AI/天
   - 付费用户：100张AI/天
   - 可购买额外配额

**测试用例:**
- TC-COST-001: 缓存有效性测试（7天）
- TC-COST-002: 缓存命中率测试
- TC-COST-003: 成本上限触发测试
- TC-COST-004: 用户配额测试

### 2.3 风险3：视频上传安全漏洞

**严重程度:** 🔴 P0 - CRITICAL

**问题描述:**
- Phase 3 PRD允许500MB视频上传
- PRD缺少文件类型验证（Magic Number）要求
- 安全风险：
  - 恶意视频上传病毒
  - MP4/MOV文件嵌入恶意代码
  - 视频文件巨大（存储成本、处理成本）
- FFmpeg服务器可能被攻击

**当前问题:**
- ❌ 5层验证不足
- ❌ 缺少Magic Number验证
- ❌ 无视频内容扫描
- ❌ 无文件大小限制

**工作量:** 20小时（修复）+ 延后Phase 4

**解决方案（P0优先级）：**
1. ✅ 五层验证
   - MIME类型验证
   - Magic Number检测（文件头）
   - 文件大小验证（最大100MB）
   - 视频尺寸验证（最大1920x1080）
   - 视频时长验证（最长5分钟）
2. ✅ 恶意内容检测
   - 集成病毒扫描API（ClamAV等）
   - 上传后异步扫描
3. ✅ FFmpeg沙箱隔离
   - 容器化处理
   - 资源限制（CPU/内存）
   - 超时杀毒进程
4. ✅ 文件大小限制
   - 最大100MB（降低存储成本）
   - 超大文件直接拒绝

**测试用例:**
- TC-VIDEO-001: 五层验证测试
- TC-VIDEO-002: Magic Number验证测试
- TC-VIDEO-003: 恶意文件上传测试
- TC-VIDEO-004: 文件大小限制测试
- TC-VIDEO-005: 病毒文件处理测试

### 2.4 风险4：访问密码强度不足

**严重程度:** 🟡 P1 - HIGH

**问题描述:**
- 当前仅支持6位数字密码
- 6位数字：1,000,000种可能
- GPU破解：1秒/密码
- 暴力破解：100次/秒

**当前问题:**
- ❌ 6位数字太弱
- ❌ 无复杂字符要求
- ❌ 暴力保护不足

**工作量:** 4小时

**解决方案（P1优先级）：**
1. ✅ 密码要求升级
   - 最少8位
   - 必须包含：大写字母 + 小写字母 + 数字
   - 禁止常见弱密码
2. ✅ 暴力保护加强
   - 5次错误锁定30分钟（当前）
   - 改为3次错误锁定30分钟
   - IP级别的限流

**测试用例:**
- TC-PWD-001: 8位字母数字密码测试
- TC-PWD-002: 暴力保护测试（3次锁定）
- TC-PWD-003: IP限流测试
